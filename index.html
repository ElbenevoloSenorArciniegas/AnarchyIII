Genial, ya generé el código ¿y ahora qué?

Hay 4 pequeñísimos pasos que debes realizar en tu código generado para que pueda funcionar:

1. Propiedades de base de datos
back/dao/properties

Este archivo es importante en extremo. En él deben colocarse los datos de acceso a cada base de datos que vaya a utilizar.
DB_NAME debe ser cambiado por (¡adivinaste!) el nombre de la base de datos. Recuerde que debe coincidir con el utilizado en el controlador (paso 2).
host debe tener la url donde se aloja la base de datos, incluyendo el puerto.
username contendrá el nombre de acceso a la base de datos, y password (¡adivinas otra vez!) la contraseña para el mismo fin.

2. Controlador global
back/facade/globalController

Define nuevamente el nombre de la base de datos y el gestor de conexión a la misma en los campos DEFAULT_DBNAME y DEFAULT_GESTOR
Actualmente Anarchy soporta las constantes:

  • MYSQL_FACTORY
  • POSTGRESQL_FACTORY
  • ORACLE_FACTORY

3. Ruta del archivo properties
back/Dao/conexion
Sólo necesario para proyectos Java web.

Define la ruta absoluta del archivo properties.ini (sí, el del primer paso) dentro del método leerProperties() para que pueda ser leido, ya que la ruta relativa puede fallar en tiempo de ejecución.
